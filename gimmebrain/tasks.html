Task Monitor

Every automated task that runs on the Good Results back end. These run on Netlify servers on a schedule — they do not require anyone to be logged in, any computer to be on, or any manual trigger. When something goes wrong, they post to Slack. When everything is fine, they log quietly and move on.

How This Works

Each task is a serverless function deployed on Netlify with a cron schedule attached to it. Netlify runs the function automatically at the scheduled time, every day or every week, regardless of whether anyone is logged in or any computer is turned on.

When a task runs, it does its job (check KBs, pull leaderboard data, scan for deal deadlines), logs the result to this page, and posts to Slack if there is anything the team needs to see. If a task fails, it posts an error alert to Slack and tags Gayden and Joe.

To add a new scheduled task, create a new Netlify function, add a schedule entry in netlify.toml, and register it in task-log.json. It deploys automatically with the next GitHub push.
